#Здравствуй, дорогой друг!

Данный проект под названием todolist является веб-приложением и выполняет функции планировщика задач.
В нем реализован весь функционал согласно ТЗ, в т.ч. задачи со *.


#Тестирование

Чтобы провести тестирование выполни следующую команду в терминале находясь в корневой директории проекта:
go test ./tests


#Сборка и запуск приложения

Чтобы собрать образ выполни следующую команду в терминале:
docker build -t todolist .

В файле .env используются следующие переменные окружения:
TODO_PORT - содержит значение порта по которому приложение будет доступно внутри контейнера (может быть пустым);
TODO_DBFILE - содержит значение имени файла базы данных и пути к нему (может быть пустым);
TODO_PASSWORD - содержит значение пользовательского пароля (может быть пустым);
TODO_SECRET - содержит значение, которое используется для генерации JWT-токена (может быть пустым).

Чтобы запустить контейнер с приложением выполни следующую команду в терминале:
docker run --env-file .env -p <1>:<2> todolist

Теперь, чтобы начать пользоваться приложением необходимо перейти по ссылке:
http://localhost:<1>/login.html

<1> - внешний порт контейнера, можно выбрать любой незанятый порт;
<2> - внутренний порт контейнера, должен соответствовать TODO_PORT, если TODO_PORT не указан при запуске контейнера стоит использовать 7540 или флаг -P, но без указания портов <1> и <2> (в этом случае порт <1> будет назначен автоматически, порт <2> - 7540).
